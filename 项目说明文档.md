## 一、项目概述

这是一个「测一测身价」职场价值评估系统，用户上传简历(CV)或输入岗位职责(JD)，系统通过 AI + HAY 评估模型计算职级和薪酬范围。

### 核心评估流程

```
用户输入 (简历/JD)
    ↓
1. 文本解析 (支持 PDF)
    ↓
2. LLM 提取关键因素 (DeepSeek API)
    ↓
3. HAY 8因素评估 → 职级计算
    ↓
4. 8因素 → 5能力映射 (雷达图)
    ↓
5. 薪酬查询 (城市/行业/职能)
    ↓
6. 生成趣味标签 + AI深度评估
    ↓
返回结果
```

### 商业模式

| 内容 | 免费 | 付费 |
|------|:----:|:----:|
| 薪酬范围 | ✓ | ✓ |
| 职级 & 趣味标签 | ✓ | ✓ |
| 5能力雷达图 | - | ✓ |
| 能力详解 | - | ✓ |
| AI深度评估 | - | ✓ |

### 各文件详细说明

#### 入口 & 配置

| 文件 | 功能 | 关键接口 |
|------|------|----------|
| **mini_api.py** | Flask API 主入口，处理评估请求 | `POST /api/mini/assess` - 核心评估接口<br>`GET /api/health` - 健康检查 |
| **config.py** | 配置管理，读取环境变量和.env文件 | `config.DEEPSEEK_API_KEY` - API密钥<br>`config.SALARY_CSV_PATH` - 薪酬数据路径 |

---

#### LLM 服务层

| 文件 | 功能 | 关键函数/类 |
|------|------|-------------|
| **llm_service.py** | DeepSeek API 封装，包含所有LLM调用逻辑 | `LLMService` 类<br>`extract_pk_and_rscp()` - 解析JD/CV提取HAY 8因素<br>`generate_deep_insight()` - 生成AI发展建议 |

**小程序使用的两个LLM场景：**
1. **解析JD/CV** → `extract_pk_and_rscp()` - 通过增量收敛算法提取8因素
2. **发展建议** → `generate_deep_insight()` - 生成AI深度评估文案

---

#### 核心计算层

| 文件 | 功能 | 关键函数/类 |
|------|------|-------------|
| **incremental_convergence.py** | 增量收敛引擎，1次LLM调用生成30-80个方案 | `IncrementalConvergence` 类<br>`find_optimal_solution()` - 查找最优解 |
| **calculator.py** | HAY三维度八因素计算 | `HayCalculator` 类<br>`calculate_hay_evaluation()` - 返回总分、职级 |
| **profile_calculator.py** | 岗位特性计算 (P型/L型/A型) | `calculate_job_profile()` |
| **salary_calculator.py** | 薪酬分位值查询 (P25/P50/P75) | `SalaryCalculator` 类<br>`get_salary_range()` - 查询薪酬区间 |
| **ability_mapper.py** | 8因素→5能力映射 | `map_hay_to_5_abilities()` - 核心映射<br>`get_ability_radar_data()` - 雷达图数据 |
| **level_tags.py** | 趣味标签生成 (搬砖圣手、六边形战士等) | `get_level_tag_and_desc()` |

**5能力映射公式：**
```
专业力 = PK(60%) + TE(40%)
管理力 = MK(70%) + FTA(30%)
合作力 = Comm(80%) + NI(20%)
思辨力 = TC(60%) + TE(40%)
创新力 = TC(70%) + M(30%)
```

| **profile_norm_validator.py** | 验证是否符合职能常模 | `ProfileNormValidator` 类 |

### API 接口说明

#### `POST /api/mini/assess`

**请求体：**
```json
{
  "assessmentType": "CV",        // "CV" 或 "JD"
  "city": "上海",                // 城市名
  "industry": "互联网",          // 行业
  "jobTitle": "产品经理",        // 职位名称
  "jobFunction": "产品",         // 职能类型
  "resumeText": "简历内容...",   // CV模式必填
  "jdText": "岗位职责..."        // JD模式必填
}
```

**响应体：**
```json
{
  "success": true,
  "data": {
    "salaryRange": "25万-35万",      // 薪酬范围
    "level": 14,                     // 职级
    "levelTag": "业务脊梁",          // 趣味标签
    "levelDesc": "你是团队的中流砥柱...", // 标签描述
    "abilities": {                   // 5能力详情 (付费)
      "专业力": {"score": 75, "level": "high", "explanation": "..."},
      "管理力": {"score": 60, "level": "medium", "explanation": "..."},
      ...
    },
    "radarData": {"专业力": 75, ...}, // 雷达图数据 (付费)
    "abilitySummary": "...",          // 能力总结 (付费)
    "deepInsight": "..."              // AI深度评估 (付费)
  }
}
```

## 四、改造建议

### 后端改造

1. **部署方式**：Flask 可直接部署，或改为云函数
2. **PDF解析**：小程序端可先用 wx.chooseMessageFile 选文件，上传到后端解析
3. **API鉴权**：增加微信登录态验证

### 前端改造

1. **框架选择**：
   - 原生小程序
   - Taro (React语法)
   - uni-app (Vue语法)

2. **重点页面**：
   - 表单页 (App.tsx renderFormContent)
   - 结果页 (ResultView.tsx)
   - 雷达图 (需替换 recharts)

3. **支付对接**：
   - 替换模拟支付为微信支付 API
   - `wx.requestPayment()`

4. **分享功能**：
   - 替换 html2canvas 为小程序 canvas
   - `wx.showShareMenu()` / `onShareAppMessage()`
